## Settings Overview

The **Settings** section is accessible from the sidebar. It has four sub-pages for managing your Elmo configuration.

## Brand Settings

Configure your brand identity:

- **Name** — Your brand name as tracked in AI responses
- **Website** — Primary website URL
- **Domains** — Additional domains associated with your brand (e.g. blog subdomain, product domains)
- **Aliases** — Alternative names or spellings that AI models might use

## Prompt Settings

Manage your tracking prompts:

- **Add prompts** — Create new prompts manually or re-run the wizard
- **Edit prompts** — Modify prompt text and tags
- **Enable/Disable** — Toggle tracking for individual prompts without deleting them
- **Delete** — Remove prompts you no longer need

## LLM Settings

The **LLMs** page controls which AI platforms Elmo tracks for this brand. Check a platform to start tracking it or uncheck it to stop — the change takes effect on the next scheduled run. Each row shows where its value comes from (your plan, an instance default, or your own pick), and why a platform isn't running when it's unavailable.

- The list of available platforms comes from your deployment's catalog, which admins manage under **Admin → Config** — not from this page or from `SCRAPE_TARGETS`.
- On cloud plans the number of platforms you can track is capped by your plan, and the page shows how many picks you have left.
- On local and cloud deployments, individual prompts can additionally turn specific models on or off (for example, adding a base or web-search Claude run) from the prompt's own page.

## Admin Configuration

Instance admins get an **Admin → Config** section for deployment-wide run configuration that members don't edit: the platform catalog (which providers and models exist), provider credentials, and default run cadence. On cloud, admins also set per-organization plan limits here. These surfaces are read-only in the demo.

## Updating Environment Variables

Some settings (like provider API keys) can be configured via environment variables; keys can also be stored encrypted in the app from the admin Providers surface. To update the environment:

```bash
# Edit your .env file in the config directory
nano ./elmo/.env

# Restart to pick up changes
elmo compose down
elmo compose up -d
```

See the [Configuration](/docs/developer-guide/configuration) page for a complete reference of environment variables.
