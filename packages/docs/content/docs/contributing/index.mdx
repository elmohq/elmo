Elmo is an open source project and we welcome contributions. Whether you're fixing a bug, adding a feature, or improving documentation, this guide will help you get started.

For the full contributor guide — including coding guidelines and security
reporting — see [`CONTRIBUTING.md`](https://github.com/elmohq/elmo/blob/main/CONTRIBUTING.md)
in the repository.

## How to Contribute

1. **Fork the repository** on GitHub
2. **Create a branch** for your changes
3. **Set up your development environment** — see [Development Setup](/docs/contributing/development-setup)
4. **Sign the CLA** if this is your first contribution (see above)
5. **Make your changes** and test them locally
6. **Add a changeset** describing your changes:
   ```bash
   pnpm changeset
   ```
7. **Commit your changes** along with the generated changeset file
8. **Open a pull request** against the `main` branch

A maintainer will review your PR. Once merged, a maintainer will handle the release process.

## What to Work On

- Check the [GitHub Issues](https://github.com/elmohq/elmo/issues) for open tasks
- Bug fixes are always welcome
- Feature requests should be discussed in an issue before implementation
- Documentation improvements are appreciated

## Guidelines

- Follow the existing code style — Biome handles linting and formatting
- Write tests for new features when applicable
- Keep PRs focused — one feature or fix per PR
- Read the [AI Contribution Policy](/docs/contributing/ai-policy) if using AI tools
