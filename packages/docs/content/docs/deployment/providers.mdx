## Quick Start

The fastest way to get started is with [Olostep](https://olostep.com), which can scrape ChatGPT, Google AI Mode, and Copilot — the three most valuable models for visibility tracking.

Set a single environment variable:

```
OLOSTEP_API_KEY=your-key-here
SCRAPE_TARGETS=chatgpt:olostep:online,google-ai-mode:olostep:online,copilot:olostep:online
```

This tells Elmo to track three models, all using Olostep as the scraping provider, all with web search enabled.

## SCRAPE_TARGETS Format

The `SCRAPE_TARGETS` environment variable is a comma-separated list of scrape targets. Each target follows the format:

```
model:provider[:version][:online]
```

| Position | Required | Description |
|----------|----------|-------------|
| 1 — `model` | Yes | AI model to track (`chatgpt`, `google-ai-mode`, `copilot`, etc.) |
| 2 — `provider` | Yes | How to reach it (`olostep`, `brightdata`, `direct`, `openrouter`, `dataforseo`) |
| 3 — `version` | For `direct` and `openrouter` | Model version slug (e.g. `gpt-5-mini`, `anthropic/claude-sonnet-4`). May contain colons for OpenRouter variants. |
| 4 — `:online` | No | Append to enable web search. Omit for no web search. |

### Examples

```bash
# Scraping providers (no version needed)
chatgpt:olostep:online
google-ai-mode:brightdata:online

# Direct API (version required)
chatgpt:direct:gpt-5-mini:online
claude:direct:claude-sonnet-4

# OpenRouter (version required, may include variant suffix)
chatgpt:openrouter:openai/gpt-5-mini:online
perplexity:openrouter:perplexity/sonar-pro:online
```

## What to Track

### Essential (Top 3)

These models cover the vast majority of AI-assisted search traffic and should be tracked by every deployment:

- **ChatGPT** with web search — the largest AI assistant by usage. Use `:online` to match the real user experience.
- **Google AI Mode** — Google's AI-generated answers are shown to billions of search users. Always uses web search.
- **Copilot** — Microsoft's AI assistant, integrated into Bing and Edge. Use `:online`.

### Sometimes Worth Tracking

- **Perplexity** — a popular AI search engine. Lower volume than the top 3, but growing. Inexpensive via OpenRouter.

### Usually Not Worth the Cost

- **Claude** — expensive, especially with web search enabled (~$0.05/query). Most users interact with Claude for writing/coding, not web search.
- **Gemini** — largely redundant with Google AI Mode for visibility tracking purposes.
- **Grok** — low market share. Track only if your audience skews toward X/Twitter users.

## Provider Details

### Olostep

**Best for**: Most deployments. Broad model support, simple setup.

- Supports: ChatGPT, Google AI Mode, Google AI Overview, Gemini, Copilot, Perplexity, Grok
- Requires: `OLOSTEP_API_KEY`
- Scrapes the actual web UI, so results match the real user experience
- Credit-based pricing (3–5 credits per query depending on model)

### BrightData

**Best for**: High-volume deployments that need per-model dataset control.

- Supports: ChatGPT, Google AI Mode, Google AI Overview, Gemini, Copilot, Perplexity, Grok
- Requires: `BRIGHTDATA_API_TOKEN` and dataset IDs passed as the version slug in `SCRAPE_TARGETS` (e.g., `chatgpt:brightdata:gd_abc123:online`)
- Uses BrightData's Web Scraper API with model-specific datasets
- More configuration overhead but competitive per-query pricing

### OpenRouter

**Best for**: Accessing multiple AI models through a single API key.

- Supports: ChatGPT, Claude, Perplexity, Gemini, Grok (and any model on OpenRouter)
- Requires: `OPENROUTER_API_KEY`
- Calls model APIs, not web UIs — results may differ from what end users see
- Web search available via the `:online` version variant
- Default versions: `openai/gpt-5-mini`, `anthropic/claude-sonnet-4`, `perplexity/sonar-pro`, `google/gemini-2.5-flash`, `x-ai/grok-3-mini`

### Direct APIs

**Best for**: When you already have API keys and want the lowest latency.

- **Direct OpenAI** (`direct` or `direct-openai`): Requires `OPENAI_API_KEY`. Supports ChatGPT model only.
- **Direct Anthropic** (`direct-anthropic`): Requires `ANTHROPIC_API_KEY`. Supports Claude model only.
- Lowest latency since there's no intermediary
- Web search is supported on both (OpenAI via `web_search_preview` tool, Anthropic via `web_search` tool)

### DataForSEO

**Best for**: Google AI Mode tracking specifically, or if you already use DataForSEO for SEO data.

- Supports: Google AI Mode only
- Requires: `DATAFORSEO_LOGIN` and `DATAFORSEO_PASSWORD`
- Uses DataForSEO's SERP API for structured AI Mode results
- Does not support web search toggle (always includes search context)

## Web Search Configuration

The `:online` suffix in `SCRAPE_TARGETS` enables web search for that target. This is important because AI models behave differently with and without web access:

- **With web search** (`:online`): The model searches the web before answering. Results include citations and reflect what users see when they ask questions in the real product. **Recommended for visibility tracking.**
- **Without web search**: The model answers from its training data only. Cheaper, but doesn't represent the real user experience for most models.

### Recommendations Per Model

| Model | Web Search Recommended? | Notes |
|--------|------------------------|-------|
| ChatGPT | Yes | Most users have web search enabled by default |
| Google AI Mode | Always on | AI Mode inherently uses web search |
| Copilot | Yes | Bing-powered web search is core to Copilot |
| Perplexity | Yes | Perplexity is fundamentally a search engine |
| Claude | Optional | Claude is often used without web search; track with `:online` only if you want to measure web-search-mode visibility |
| Gemini | Yes | Most Gemini users interact via Google, which includes search |
| Grok | Optional | Depends on your audience |

For scraping providers (Olostep, BrightData), web search is inherent in the scrape — the `:online` flag is still accepted for consistency but the scrape always reflects the real UI which includes search.

## Brand-Level Overrides

By default, every brand in your workspace is evaluated against all configured `SCRAPE_TARGETS` models. You can narrow this per brand using the `enabled_models` column on the brands table.

When `enabled_models` is set for a brand (e.g. `["chatgpt", "google-ai-mode"]`), only those models will be used for that brand's prompts. When `NULL` (the default), all configured models apply.

This is useful when:
- Different brands operate in different markets where certain models matter more
- You want to reduce costs by only tracking the most relevant models per brand
- Some brands are experimental and only need limited tracking
